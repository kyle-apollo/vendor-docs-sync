1. Overview

The VendorX Data Pipeline Connector is a lightweight agent that forwards structured event streams to the VendorX ingest endpoint. It supports batching, retry logic, and optional local buffering.

This guide covers a fresh install on Linux or macOS. For container deployments see the Kubernetes guide in the VendorX portal.

2. Prerequisites

RequirementDetails
Operating systemLinux (glibc ≥ 2.17) or macOS 12+
NetworkOutbound HTTPS (443) to ingest.vendorx.example.com
API keyProvisioned via the VendorX portal; requires ingest:write
ConfigYAML config file (see Section 5)
Note If outbound 443 is blocked, installation may still succeed but the smoke test will fail.

3. Installation

1 Download the agent binary

curl -fsSL "https://packages.vendorx.example.com/agent/VERSION/vendorx-agent-PLATFORM" \
  -o /tmp/vendorx-agent

2 Verify the checksum

echo "EXPECTED_CHECKSUM  /tmp/vendorx-agent" | sha256sum -c -

3 Install the binary

chmod +x /tmp/vendorx-agent
sudo mv /tmp/vendorx-agent /usr/local/bin/vendorx-agent
vendorx-agent --version

4. Configuration

4.1 YAML config

vendorx:
  endpoint: https://ingest.vendorx.example.com/v2/events
  api_key: ${VENDORX_API_KEY}
  batch_size: 100
  flush_interval_seconds: 5
  retry:
    max_attempts: 3
    backoff_seconds: 2
  tls:
    verify: true
Warning Never hard-code the API key in a committed config file.

4.2 Environment variables

VariableRequiredDescription
VENDORX_API_KEYYesAPI key with ingest:write scope
APP_ENVNodevelopment / staging / production
LOG_LEVELNoDEBUG / INFO / WARN / ERROR

5. First run & smoke test

vendorx-agent start --config sample_project/app_config.yaml --foreground

7. Troubleshooting

Connection refused / TLS handshake failure

  • Verify outbound port 443 is open: curl -v https://ingest.vendorx.example.com/health
  • Confirm system time is accurate: date -u
  • Run the Support Ticket Macro in Section 6 and attach the output.

401 Unauthorized

  • Confirm VENDORX_API_KEY is set where the agent runs.
  • Verify the key has not expired and has ingest:write.

8. Contacting support

Open a ticket at https://support.vendorx.example.com and include the diagnostics bundle from Section 6.
